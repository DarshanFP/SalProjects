## 1. Role Overview

### 1.1 Your Role

As an **Admin**, you have the highest level of access in the system. You are responsible for:

-   **Full System Access:** Access to all routes, features, and data across the entire system
-   **System Configuration:** System-wide settings and configuration
-   **User Management:** Create and manage users across all roles
-   **System Oversight:** Monitor system performance, health, and operations
-   **Troubleshooting:** Resolve system-wide issues and provide technical support

### 1.2 Access Level

-   **Complete System Access:** Access to ALL routes and features (catch-all route)
-   **All User Roles:** Can access functionality of all roles (Admin, General, Coordinator, Provincial, Executor, Applicant)
-   **System Configuration:** System settings and configuration access
-   **Full Data Access:** Access to all data across all provinces, centers, users, projects, and reports
-   **No Restrictions:** No role-based restrictions on your access

### 1.3 Key Characteristics

-   **Top-Level Access:** Highest privilege level in the system
-   **Universal Access:** Can perform any action any other role can perform
-   **System Administrator:** Technical and administrative oversight
-   **Catch-All Route:** Has access to all routes via admin catch-all route

## 2. Getting Started

### 2.1 Logging In

1. Navigate to the application login page
2. Enter your **email address** and **password**
3. Click the **Login** button
4. You will be redirected to your Admin dashboard

### 2.2 First Login Steps

1. **Change Your Password** (Highly Recommended)

    - Click on your profile name in the top-right corner
    - Select **Profile** or **Change Password**
    - Enter your current password and new password
    - Click **Update Password**

2. **Familiarize Yourself with the System**
    - Explore the dashboard
    - Review system structure
    - Understand user roles and hierarchy
    - Review available features

## 4. System Access

### 4.1 Accessing All Features

As Admin, you have access to:

1. **All User Role Features:**

    - All Coordinator features (project/report approval, provincial management)
    - All Provincial features (team management, report forwarding)
    - All Executor/Applicant features (project creation, reporting)
    - All General features (coordinator management, province management)

2. **All Routes:**

    - Access all routes in the system
    - Catch-all route ensures access even if routes change
    - Can test functionality from any user role perspective

3. **All Data:**
    - View all projects across the system
    - View all reports across the system
    - View all users and their information
    - View all budgets and expenses
    - View all activities and logs

### 4.2 Testing Features

You can test features from different user role perspectives:

-   Access Coordinator dashboard to test coordinator features
-   Access Provincial dashboard to test provincial features
-   Access Executor dashboard to test executor features
-   This helps with troubleshooting and system validation

## 5. User Management

### 5.1 Creating Users

You can create users for any role:

**Steps:**

1. Navigate to user management (if available) or use appropriate role-specific creation routes
2. Select the role (Admin, General, Coordinator, Provincial, Executor, Applicant)
3. Fill in user information:
    - Name, Email, Password
    - Role-specific fields (Province, Center, etc.)
    - Status (Active/Inactive)
4. Save user account

### 5.2 Managing Users

You can:

-   **Edit Users:** Update user information for any role
-   **Activate/Deactivate:** Manage user access across the system
-   **Reset Passwords:** Reset passwords for any user
-   **View User Details:** View complete user information
-   **Monitor User Activity:** View user activity and history

### 5.3 User Access Management

-   Manage user status (Active/Inactive)
-   Reset user passwords
-   Update user information
-   Assign users to appropriate roles and hierarchies

## 8. Profile Management

### 8.1 Viewing Your Profile

**Steps:**

1. Click on your name/profile picture in the top-right corner
2. Select **"Profile"**
3. View your profile information:
    - Name, Email, Role (Admin)
    - Other profile details

### 8.2 Updating Profile Information

**Steps:**

1. Go to Profile page
2. Click **"Edit"** button
3. Update the fields you want to change
4. Click **"Update Profile"** to save changes

### 8.3 Changing Password

**Steps:**

1. Go to Profile page
2. Click **"Change Password"**
3. Enter:
    - Current Password
    - New Password
    - Confirm New Password
4. Click **"Update Password"**
5. You will be logged out and need to login with new password

## 9. Troubleshooting

### 9.1 Common Issues

**Issue: Need to test a feature from a specific user role perspective**

-   **Solution:** Access the appropriate dashboard (Coordinator, Provincial, Executor) to test features from that role's perspective

**Issue: User cannot access system**

-   **Solution:** Check user status (should be Active), verify email is correct, reset password if needed

**Issue: System performance issues**

-   **Solution:** Review system logs, check system health indicators, investigate performance metrics

**Issue: Data inconsistencies**

-   **Solution:** Review data integrity, check audit trails, investigate specific data issues

### 9.2 Getting Help

-   Review system documentation
-   Check system logs for errors
-   Review activity history for insights
-   Contact technical support if needed

### 9.3 Best Practices

1. **Security:** Use strong passwords, change passwords regularly
2. **Access Control:** Only grant admin access to trusted users
3. **Monitoring:** Regularly monitor system health and performance
4. **Documentation:** Document system changes and configurations
5. **Backup:** Ensure system backups are in place
6. **Testing:** Test features from different user role perspectives before deployment
7. **User Support:** Provide support to users when needed
8. **System Maintenance:** Perform regular system maintenance

## Appendix B: User Role Hierarchy Reference

### System Hierarchy

1. **Admin** (You) - Full system access
2. **General** - Coordinator access + direct team management + province management
3. **Coordinator** - Manages Provincials, final approval authority
4. **Provincial** - Manages Executors/Applicants, forwards to Coordinator
5. **Executor/Applicant** - Creates projects, submits reports

### Your Position

-   **Top Level:** Highest privilege level
-   **Universal Access:** Can access any level
-   **No Restrictions:** No role-based limitations
